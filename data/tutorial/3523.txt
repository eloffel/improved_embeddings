how to get started in nlp

   [16]go to the profile of melanie tosik
   [17]melanie tosik (button) blockedunblock (button) followfollowing
   may 1, 2017
   [1*cioj1mkzf6qzexqkupvfwg.png]
   dependency parse tree visualized by [18]displacy

   somewhere i read that if you ever have to answer the same question
   twice, it   s probably a good idea to turn it into a blog post. in
   keeping with this rule and to save my future self some time, here now
   my standard answer to the question:    my background is in * science, and
   i   m interested in learning nlp. where do i start?   

   before you dive in, please note that the list below is really just a
   very general starting point (and likely incomplete). to help navigate
   the flood of information, i added short descriptions and difficulty
   estimates in brackets. basic programming skills (e.g. in python) are
   recommended.
     __________________________________________________________________

online courses

     * [19]dan jurafsky & chris manning: natural language processing
       [great intro video series]
     * [20]stanford cs224d: deep learning for natural language processing
       [more advanced ml algorithms, deep learning, and nn architectures
       for nlp]
     * [21]coursera: introduction to natural language processing [intro
       nlp course offered by the university of michigan]

libraries and open source

     * spacy ([22]website, [23]blog) [python; emerging open-source library
       with [24]fantastic usage examples, api documentation, and [25]demo
       applications]
     * natural language toolkit (nltk) ([26]website, [27]book) [python;
       practical intro to programming for nlp, mainly used for teaching]
     * stanford corenlp ([28]website) [java; high-quality analysis
       toolkit]
     * allennlp ([29]website) [python; nlp research library built on
       [30]pytorch]
     * fasttext ([31]website) [c++; efficient library for text
       classification and representation learning]

active blogs

     * [32]natural language processing blog (hal daum   iii)
     * [33]language log (mark liberman)
     * [34]google research blog
     * [35]explosion ai blog
     * [36]hugging face
     * [37]sebastian ruder   s blog

books

     * [38]speech and language processing (jurafsky and martin) [classic
       nlp textbook that covers all the basics, 3rd edition coming soon]
     * [39]foundations of statistical natural language processing (manning
       and sch  tze) [more advanced, statistical nlp methods]
     * [40]introduction to information retrieval (manning, raghavan and
       sch  tze) [excellent reference on ranking/search]
     * [41]neural network methods in natural language processing
       (goldberg) [deep intro to nn approaches to nlp, [42]primer here]
     * [43]linguistic fundamentals for natural language processing
       (bender) [morphology and syntax basics for more successful nlp]
     * [44]deep learning (goodfellow, courville and bengio) [best intro to
       deep learning]

miscellaneous

     * [45]how to build a id97 model in tensorflow [tutorial]
     * [46]deep learning for nlp resources [overview of state-of-the-art
       resources for deep learning, organized by topic]
     * [47]last words: computational linguistics and deep learning         a
       look at the importance of natural language processing. (manning)
       [article]
     * [48]natural language understanding with distributed representation
       (cho) [self-contained lecture note on ml/nn approaches to nlu]
     * [49]bayesian id136 with tears (knight) [tutorial workbook]
     * [50]association for computational linguistics (acl) [journal
       anthology]
     * [51]quora: how do i learn natural language processing?
     * [52]natural language understanding and computational semantics
       (bowman) [open-source course syllabus with comprehensive slides]
     * [53]fast.ai [   making neural nets uncool again   ]

diy projects and data sets

   [1*itvqqiu6cfdpq33b6rocdw.png]
   source: [54]http://gunshowcomic.com/

   a thorough [55]list of publicly available nlp data sets has already
   been created by nicolas iderhoff. beyond these, here are some projects
   i can recommend to any nlp novice wanting to get their hands dirty:
     * implement a [56]part-of-speech (pos) tagger based on a [57]hidden
       markov model (id48)
     * implement the [58]cyk algorithm for parsing [59]context-free
       grammars
     * implement [60]semantic similarity between two given words in a
       collection of text, e.g. [61]pointwise mutual information (pmi)
     * implement a [62]naive bayes classifier to [63]filter spam
     * implement a [64]spell checker based on [65]id153s between
       words
     * implement a [66]markov chain text generator
     * implement a [67]topic model using [68]id44
       (lda)
     * use [69]id97 to generate id27s from a large text
       corpus, e.g. [70]wikipedia
     * use [71]id116 to cluster [72]tf-idf vectors of text, e.g. news
       articles
     * implement a [73]named-entity recognizer (ner) (also called a name
       tagger), e.g. following the [74]conll-2003 shared task
